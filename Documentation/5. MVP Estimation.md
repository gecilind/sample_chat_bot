# MVP Estimation — Infleet AI Support Agent (Corrected Stack)

---

## Phase 1 — Web Chat MVP

| # | Milestone | Stack | Days | What It Does |
|---|---|---|---|---|
| 1 | **Knowledge Base (pgvector + Ingestion)** | Supabase PostgreSQL + pgvector + OpenAI Embeddings API | 3 Days | Chunk manuals, generate embeddings via OpenAI, store in pgvector with HNSW index. Build `/ingest-manual` and `/search-kb` routes. |
| 2 | **Backend Core (FastAPI Server)** | Python + FastAPI + uvicorn + asyncpg | 5 Days | All server-side logic: `/chat`, `/create-ticket`, `/search-kb` routes. Async Postgres connections to our Supabase DB. |
| 3 | **AI Agent (Chat Reasoning)** | OpenAI Assistants API (direct) | 4 Days | Configure Assistant with system prompt, function calling (KB search, ticket creation), intent classification, and conversation thread management. |
| 4 | **User Context Integration** | Read Infleet's session (`window.__INFLEET_USER__`) | 1 Day | React reads injected user context on mount. No auth UI on our side — Infleet's portal handles all login. We pass `userId` and `email` to backend. Device identification is handled by the AI in conversation. |
| 5 | **Chat UI** | Custom React Component (Vite + TypeScript) | 3 Days | Embeddable chat widget with full branding control. All API calls via native `fetch()` to our FastAPI backend. Conversation state, message rendering, loading states. |
| 6 | **Jira Integration** | `httpx` call from `/create-ticket` FastAPI route | 2 Days | Direct REST API call to Infleet's Jira. Structures support ticket fields from AI output (summary, description, labels, priority). No SDK, no middleware. |
| 7 | **Testing & Integration** | End-to-end flow validation | 3 Days | Full path testing: FAQ resolution via RAG, escalation to Jira ticket, error handling, edge cases, conversation history persistence. |
| 8 | **Sprint Management** | Lean 2-person sprint (Linear / Notion) | 1 Day | Minimal overhead — daily syncs, task tracking, deployment pipeline setup (Railway / Render / VPS). |

**Total — Phase 1: 22 Days**

---

## Phase 2 — Voice & Multimodal MVP

| # | Milestone | Stack | Days | What It Does |
|---|---|---|---|---|
| 10 | **Voice Relay Backend** | FastAPI WebSocket `/voice-relay` route | 3 Days | WebSocket endpoint that relays PCM16 audio frames between browser and OpenAI Realtime API. Bidirectional stream, session management, error recovery. |
| 11 | **OpenAI Realtime API Integration** | OpenAI Realtime API (WebSocket) | 3 Days | Configure Realtime API session: STT → reasoning → function calling → TTS in a single stream. Same tools as chat (KB search, tickets). |
| 12 | **Voice UI (Browser)** | Custom React Voice Widget + AudioWorklet | 3 Days | `getUserMedia()` for mic access, AudioWorklet for PCM16 encoding, WebSocket to `/voice-relay`, AudioContext for playback. Visual feedback (recording state, speaking state). |
| 13 | **Voice ↔ Chat Convergence** | Shared backend logic (FastAPI) | 2 Days | Ensure voice path triggers same ticket / KB flows as chat. Unified conversation history storage. Text confirmation overlay for ticket numbers in voice mode. |
| 14 | **Testing & Polish** | Cross-browser audio testing + E2E | 1 Day | Mic permissions, audio quality, latency testing, fallback to chat if mic denied, mobile browser compatibility. |

**Total — Phase 2: 12 Days**

---

## Grand Total — Full MVP: 34 Days
